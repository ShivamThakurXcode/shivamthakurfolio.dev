Overview

A data platform needed a customer-facing dashboard that showed live product metrics without lag, even for accounts generating enormous volumes of events. I built the full stack — ingestion, aggregation, streaming and UI.

The problem

The existing tool fetched raw rows and rendered them client-side. It buckled under large accounts: charts stuttered, updates lagged by minutes, and the browser tab would freeze entirely for the biggest customers.

The solution

I moved aggregation to the server, pre-computing metrics into time buckets so the client never touches raw data. A Next.js front end renders a fast server-side shell, then a WebSocket channel streams delta updates every few hundred milliseconds. Charts are virtualized to render only the visible window, keeping frame rates high regardless of dataset size.

Aggregate on the server, virtualize on the client — that is how a dashboard stays instant at any scale.

FAQ

How did the dashboard stay fast with millions of data points?

By aggregating on the server and virtualizing on the client. The backend pre-aggregates metrics into time buckets so the browser never receives raw rows, and charts render only the visible window. This keeps the UI at interactive frame rates regardless of dataset size.

How were live updates delivered?

Over a WebSocket channel pushing delta updates every few hundred milliseconds. The client merges deltas into its local store, so charts animate smoothly without re-fetching entire datasets.

Why Next.js for an analytics product?

Next.js gave server-rendered initial loads for fast first paint, API routes for the data layer, and easy deployment. Real-time streaming ran alongside it over WebSockets, combining fast static shells with live data.
