Drone startup
SaaS platform

A full-stack platform that lets a drone startup manage its fleet, watch live telemetry, and onboard new operators in minutes instead of hours.

Overview

An early-stage drone company needed a single place to register devices, plan missions, and monitor flights in real time. I designed and built the platform end to end — from database schema to the operator-facing dashboard.

The problem

Operators were juggling spreadsheets and a legacy tool that could not show live data. Onboarding a new device took hours of manual setup, and there was no reliable way to see the status of the whole fleet at a glance.

The solution

I built a React dashboard backed by a Node.js and Express API, with MongoDB for device and mission data. Live telemetry streams over WebSockets, throttled and delta-encoded on the server so the browser only receives what changed. Redis handles caching and pub/sub so multiple dashboard sessions stay in sync. A redesigned, guided onboarding flow replaced the manual setup entirely.

The dashboard finally shows the whole fleet in real time — and new operators are productive on day one.

FAQ

What was the core technical challenge?

Streaming live telemetry from many drones to the browser without overwhelming the client or the server. The solution used a WebSocket layer with server-side throttling and delta updates, so the dashboard stayed smooth even with dozens of active devices.

What stack was used to build it?

A React front end, a Node.js and Express API, MongoDB for device and mission data, Redis for caching and pub/sub, and WebSockets for real-time telemetry. The app was deployed on a container platform behind a load balancer.

What measurable results did the project achieve?

Onboarding time dropped by 40% after redesigning the setup flow, the live dashboard sustained dozens of concurrent devices at interactive frame rates, and API p95 latency stayed under 200ms during load tests.
